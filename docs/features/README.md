# 功能规格 (Features)

> **PRD 与验收标准**  
> 本目录用于存放功能需求文档。

---

## 目录说明

本目录用于记录：

- 产品需求文档（PRD）
- 功能规格说明
- 验收标准（Acceptance Criteria）
- 用户故事（User Stories）

## 示例

待添加功能时，创建如 `user_authentication.md`：

```markdown
# 用户认证功能

## 需求概述

实现基于 JWT 的用户认证系统

## 验收标准

- [ ] 用户可以注册账号
- [ ] 用户可以登录获取 Token
- [ ] Token 有效期为 7 天
- [ ] 支持刷新 Token

## 技术方案

...
```

---

> **提示**: 功能文档应在实现前编写，作为开发和测试的依据。
