# 事故复盘 (Incidents)

> **故障档案与改进措施**  
> 本目录用于记录生产事故和复盘。

---

## 目录说明

本目录用于记录：

- 生产事故经过
- 根因分析（Root Cause Analysis）
- 改进措施
- 事故时间线

## 事故文档模板

```markdown
# 事故 #001: <简短标题>

## 基本信息

- **发生时间**: 2026-01-15 14:30 UTC+8
- **影响范围**: 全部用户 / 部分用户
- **持续时间**: 30 分钟
- **严重级别**: P0 (Critical) / P1 (High) / P2 (Medium)

## 事故经过

1. 14:30 - 发现问题
2. 14:35 - 开始排查
3. 15:00 - 恢复服务

## 根因分析

...

## 改进措施

- [ ] 措施 1
- [ ] 措施 2

## 经验教训

...
```

---

> **提示**: 事故复盘文化是工程成熟度的重要标志。
