# 存档 (Archive)

> **已废弃的设计与文档**  
> 本目录用于存放已过时但仍有参考价值的文档。

---

## 目录说明

本目录用于保存：

- 已废弃的技术方案
- 已放弃的功能设计
- 旧版本的架构文档
- 历史决策记录

## 归档规则

1. 文件命名格式：`YYYYMMDD_<原文件名>`
2. 文件顶部注明归档日期和原因
3. 提供替代方案链接（如有）

## 归档示例

```markdown
# [已废弃] 旧版认证方案

> **归档日期**: 2026-01-10  
> **废弃原因**: 改用 JWT 认证  
> **替代方案**: [新版认证](../features/user_authentication.md)

---

原文档内容...
```
